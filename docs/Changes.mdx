## 🐍 Changes in 2.9.1

:::warning

⚠️ If you are using a version lower than **2.9.0**, please refer to [upgrade guide to 2.9.0](/docs/2.9.0/changes) first, as v2.9.0 is the major release containing the braking changes. Please update your code accordingly.

# ✨ New Features

> ## New Player Option:
> - [`applyVolumeAsFilter`](./Typedefs/PlayerOptions.mdx) Allows applying volume changes as a filter for more consistent audio handling.
>
> :::warning
> In this version, the [`applyVolumeAsFilter`](./Typedefs/PlayerOptions.mdx) PlayerOption contains an error, resulting in no audio.
>
> This is already fixed in the developer version.
>
> If you want to use the fixed PlayerOption, update to the dev version:
>
> <Tabs>
>     <TabItem value="npm" label="npm" default>
>         ```sh
>         npm install magmastream@dev
>         ```
>     </TabItem>
>     <TabItem value="pnpm" label="pnpm" default>
>         ```sh
>         pnpm install magmastream@dev
>         ```
>     </TabItem>
>     <TabItem value="yarn" label="yarn" default>
>         ```sh
>         yarn add magmastream@dev
>         ```
>     </TabItem>
> </Tabs>
>
> :::

> ## Expanded [TrackSourceTypes](./Enums/TrackSourceTypes.mdx) enum
> - More sources are now supported and categorized.

> ## [PortableUser](./Typedefs/PortableUser.mdx) Interface
>  - Standardized requester object for safer and more flexible use across players and events.

---

# ⚖️ Fixes & Improvements

> ## Node Connection
> Fixed missing await call on node.connect() to ensure proper connection flow.

> ## SeyfertManager
> Added WorkerClient as a proper client instance to improve stability.

> ## Serialization
>  Switched to safe-stable-stringify for safer JSON serialization across all state storage and logging.
