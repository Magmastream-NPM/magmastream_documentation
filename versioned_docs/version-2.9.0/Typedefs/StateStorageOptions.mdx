> Represents the `stateStorageOptions` object used to configure how the manager persists player state, including Redis and JSON-based backends.
> 
> | <i className="fa-solid fa-hashtag"/> Property            | <i className="fa-solid fa-font"/> Type                                                              | <i className="fa-solid fa-list"/> Description       | <i className="fa-solid fa-flask"/> Optional |
> |:---------------------------------------------------------|:--------------------------------------------------------------------------------------------------- |:--------------------------------------------------- |:-------------------------------------------:|
> | `type`                                                   | [`StateStorageType`](../Enums/StateStorageType.mdx)                                                   | The storage method to use (`redis` or `json`).      | <i className="fa-solid fa-xmark"/>          |
> | `redisConfig`                                            | [`RedisConfig`](./RedisConfig.mdx)                                                                    | Configuration for Redis-based storage.              | <i className="fa-solid fa-check"/>          |
> | `jsonConfig`                                             | [`JsonConfig`](./JsonConfig.mdx)                                                                      | Configuration for JSON file-based storage.          | <i className="fa-solid fa-check"/>          |
> | `deleteInactivePlayers`                                  | [boolean](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Boolean) | Whether to delete inactive players when they leave. | <i className="fa-solid fa-check"/>          |
